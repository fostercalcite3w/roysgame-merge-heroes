# 遊戲開發 PLAYBOOK

每個功能從想法到交付的固定流程。不能跳步。

---

## 功能開發 SOP

```
第 1 步：理解需求
  ↓
第 2 步：實作（含觸控+鍵盤）
  ↓
第 3 步：自我 QA（跑清單，不通過不能往下）
  ↓
第 4 步：交付測試單給用戶
  ↓
第 5 步：用戶測試 + 回饋
  ↓
第 6 步：修正 + 再跑自我 QA
  ↓
第 7 步：用戶確認 OK
  ↓
第 8 步：Commit + 更新管線（如果踩到新坑）
```

### 第 1 步：理解需求

- 用戶說想要什麼
- 確認我理解正確（複述一次）
- 如果需要設計討論，先 brainstorm 再動手

### 第 2 步：實作

- 讀 CLAUDE.md 的硬規則
- UI 座標用相對比例
- 文字用繁體中文
- **鍵盤和觸控同時做**，不能拆開
- 選擇類 UI 加確認步驟

### 第 3 步：自我 QA（硬閘門）

完成實作後，**自己跑以下清單**，全部通過才能跟用戶說「可以測了」：

#### 自我 QA 清單

**基本**
- [ ] `npm test` 全部通過
- [ ] `bash scripts/verify-ui.sh` 全部通過
- [ ] dev server 啟動沒有 console error

**螢幕適配**
- [ ] 電腦瀏覽器上看起來正常
- [ ] 把瀏覽器視窗拉窄（模擬手機寬度），UI 沒有跑版
- [ ] 把瀏覽器視窗拉成直向比例，有顯示轉橫提示

**操作**
- [ ] 鍵盤操作正常
- [ ] 確認觸控操作有寫（搖桿、按鈕）

**文字**
- [ ] 畫面上所有文字都是繁體中文
- [ ] 沒有 placeholder 英文（HP、XP、Level、Click 等）

**互動**
- [ ] 選擇類 UI 有 highlight + 確認按鈕
- [ ] 按鈕/卡片點擊回饋正常（hover 變色、press 變色）

### 第 4 步：交付測試單

跟用戶說「可以測了」的時候，**附上測試單**，格式：

```
可以測了！請幫我確認這幾件事：
1. ___（具體動作）→ 預期結果
2. ___（具體動作）→ 預期結果
3. ___（具體動作）→ 預期結果
電腦和 iPhone 都測一下。
```

不能只說「試試看」。要給明確的測試項目。

### 第 5 步：用戶測試 + 回饋

- 用戶照測試單測
- 回報結果（OK 或壞了）
- 如果有額外發現也一起記錄

### 第 6 步：修正

- 修 bug
- 再跑一次第 3 步的自我 QA
- 通過後才跟用戶說「修好了」

### 第 7 步：用戶確認

- 用戶確認 OK
- 如果還有問題，回到第 6 步

### 第 8 步：Commit + 管線更新

- Commit
- 如果這次踩到新坑：
  - 更新父 repo `docs/pipeline-evolution.md`（加新教訓）
  - 更新 `CLAUDE.md`（加新硬規則，如果適用）
  - 更新 `scripts/verify-ui.sh`（加新自動檢查，如果可以自動化）

## Retro 觸發時機

以下情況必須停下來做檢討（更新父 repo 的 pipeline-evolution.md）：

1. **用戶回報 bug** — 每次都要問：管線哪一步該攔住？
2. **同一類 bug 出現第二次** — 表示上次的管線修正沒有生效，要加強
3. **完成一個 Phase** — Phase 結束時做整體回顧
4. **用戶說「檢討」** — 立刻停下來回顧

## 失敗模式速查

以下是容易犯的錯，實作時先看一眼：

| 做什麼的時候 | 容易犯的錯 | 怎麼避免 |
|---|---|---|
| 寫 UI 定位 | 寫 `400`, `300` | 用 `w * 0.5`, `h * 0.5` |
| 寫按鈕文字 | 用英文 | 全部繁體中文 |
| 加新操作 | 只做鍵盤 | 同時加觸控 |
| 選擇類 UI | 點了直接生效 | 加確認按鈕 |
| 改 Phaser config | 用 window 尺寸 | 讓 RESIZE 處理 |
| 改 CSS | 用 vh | 用 dvh |
| 做完跟用戶說 | 「試試看」 | 附測試單 |
| 修完 bug | 直接 commit | 先問管線該怎麼改 |

## Codex Plugin 工作流

### 功能開發（Claude + Codex 合作）
1. Claude 規劃設計
2. `/codex:rescue` 交給 Codex 實作
3. `/codex:review` 讓 Codex 做 code review
4. Claude 根據 review 結果修正

### Bug 修復
1. Claude 用 /investigate 找根因
2. Claude 或 Codex 修復
3. `/codex:review` 驗證修復品質

### 數值調整
1. 改 `src/data/balance.js`
2. `npm test` 驗證
3. 部署 + 玩測

### Codex sandbox 限制

Codex 無法：
- 寫入本 repo 之外的檔案（例如父 repo `.git/`）
- 連線 GitHub API（`gh` 指令會失敗）
- 跨 repo 做 git 操作

遇到這類需求時，由 Claude 直接用 Bash 執行，不要派 Codex。
